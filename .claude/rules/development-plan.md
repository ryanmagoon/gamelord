# Project Tracking

GitHub Issues are the source of truth for project state. Use parent issues with sub-issues to organize multi-part work.

- **When completing work**, close the corresponding GitHub issue via the PR (`Closes #123`).
- **When discovering new work**, create a GitHub issue. If it belongs to an existing initiative, add it as a sub-issue of the parent.
- **When encountering bugs or limitations**, create a GitHub issue with the appropriate labels.
- **For architecture context**, read `ARCHITECTURE.md`.
